初探kubernetes

实验环境：Ubuntu 18.04

实验工具：minikube、VirtualBox

实验目标：基于k8s搭建nginx负载均衡集群

一、安装minikube

1.在 BIOS 中启用 VT-x 或 AMD-v 虚拟化

2.安装VirtualBox。

注意：

Minikube 也支持 --vm-driver=none 选项，该选项在主机而非 VM 上运行 Kubernetes 组件。 使用这个驱动程序需要 Docker 和 linux 环境，而不需要 hypervisor。

3.安装kubectl

​ 4.2 启动minikube，执行 minikube start 或 minikube start --vm-driver=none

# deployment-nginx.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.17
        ports:
        - containerPort: 80

2.执行 kubectl create -f deployment-nginx.yaml 创建服务

3.执行 kubectl expose deployment nginx-deployment 将服务暴露给外部以提供访问。

也可以使用 kubectl expose deployment nginx-deployment --type=NodePort

直接指定type的值。第4步主要演示对生成service 文件的修改

4.修改以生成的service配置文件，执行 kubectl edit service nginx-deployment -o yaml，将type属性值由默认的ClusterIP改为LoadBalancer。再次查看生成的service配置文件，执行kubectl get service nginx-deployment -o yaml,获得如下配置信息

apiVersion: v1
kind: Service
metadata:
  creationTimestamp: "2020-01-06T03:30:04Z"
  labels:
    app: nginx
  name: nginx-deployment
  namespace: default
  resourceVersion: "67116"
  selfLink: /api/v1/namespaces/default/services/nginx-deployment
  uid: 423cddd5-e0e1-4f6c-873d-9267e0b72178
spec:
  clusterIP: 10.96.157.90
  externalTrafficPolicy: Cluster
  ports:
  - nodePort: 32101
    port: 80
    protocol: TCP
    targetPort: 80
  selector:
    app: nginx
  sessionAffinity: None
  type: LoadBalancer
status:
  loadBalancer: {}

Kubernetes中的Service是集群中一组Pod以及访问策略的抽象。可以通过YAML、JSON定义，目标Pods通常通过LabelSelector定义。通过type字段，服务定义了应用暴露的几种方式：

  • ClusterIP，默认的方式，通过集群IP来对外提供服务，这种方式只能在集群内部访问。
  • NodePort，利用NAT技术在Node的指定端口上提供对外服务。外部应用通过*:*的方式访问。
  • LoadBalancer，利用外部的负载均衡设施进行服务的访问。
  • ExternalName，这是1.7版本之后 kube-dns 提供的功能。

三、实验结果

1、获取服务对外访问地址，执行 minikube service nginx-deployment --url，得到访问地址http://192.168.99.100:32101

2、 浏览器打开链接，即可正常访问nginx服务
