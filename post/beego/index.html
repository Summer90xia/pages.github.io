实验环境

  • mysql 8.0

  • go1.13.5 + beego + bee

// 1 配置路由 rooters/rooter.go
func init() {
    /*
    	beego的路由配置有3钟方式
    	1 根据请求方式调用相应的方法，如 当使用 GET 请求方式时 会调用 MainController下的 Get()方法
    	2 采用注解的方式在 controller 中的方法中标明，默认采用脚手架生成的控制器即是此种方式
    	2 根据路由配置调用，即如下代码所示方式
    */
	// beego.Router("/", &controllers.MainController{})

	beego.Router("/user/:id", &controllers.UserController{}, "get:GetUser")
	beego.Router("/users", &controllers.UserController{}, "get:GetUsers")
	beego.Router("/user/add", &controllers.UserController{}, "post:CreateUser")
	beego.Router("/user/delete", &controllers.UsersController{}, "delete:DeleteUser")
}

// 2 实现controller controllers/user.go
type UserController struct {
	beego.Controller // 继承controller基类
}

func (this *UserController) GetUser() {
	// 返回模板
	this.TplName = "user/index.tpl"

	// 返回字符串
	this.Ctx.WriteString("hello beego")

	// 获取配置项值
	mu := beego.AppConfig.String("mysqluser")
	this.Ctx.WriteString(mu)

	// 返回 json
	idStr := this.Ctx.Input.Param(":id")
	id, err := strconv.ParseInt(idStr, 0, 64)
	if err != nil {
		this.Ctx.WriteString("id数据类型错误")
	}

	v, err := models.GetUserById(id)
	if err != nil {
		this.Ctx.WriteString("数据获取失败")
	}
	this.Data["json"] = v
	this.ServeJSON()
}

实验总结

bebgo的优势：

  1. bee 工具是一个为了协助快速开发 beego 项目而创建的项目，通过 bee 您可以很容易的进行 beego 项目的创建、热编译、开发、测试、和部署。

  2. 简单化：RESTful 支持、MVC 模型，可以使用 bee 工具快速地开发应用，包括监控代码修改进行热编译、自动化测试代码以及自动化打包部署。

  3. 智能化：支持智能路由、智能监控，可以监控 QPS、内存消耗、CPU 使用，以及 goroutine 的运行状况，让您的线上应用尽在掌握。

  4. 模块化：beego 内置了强大的模块，包括 Session、缓存操作、日志记录、配置解析、性能监控、上下文操作、ORM 模块、请求模拟等强大的模块，足以支撑你任何的应用。

  5. 高性能：beego 采用了 Go 原生的 http 包来处理请求，goroutine 的并发效率足以应付大流量的 Web 应用和 API 应用，目前已经应用于大量高并发的产品中。

beego的不足：

  1. 吐槽最多的就是orm这一块了
