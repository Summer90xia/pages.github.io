名哥，好久不见，最近咋样呢，听说你准备去微软呢，成功了吗？
我最近有点困惑，关于个人的职业规划问题想跟你请教一下呢。

目前我在这家公司负责国际官网的开发，形式类似于一个cms，以资讯内容为主。
用的技术仅限于 yii2 + mysql + redis，基本上不会有大数据和大并发的情况。
另外我们项目组没有专业的前端，我现在大部分时间（80%）都在写前端样式，事情多而杂，加班的情况非常多。

我感到困惑的点是，现在做的东西基本上是2年左右的经验就能完成的，且这个状态还有可能会一直持续下去，
因为公司业务主要还是做资讯的。目前团队就3个人，基本没什么技术底蕴，
之前的项目都是 PHP + MySQL 的模式，比较单薄。
我现在主要担心的是自己在这里没有发展空间，技术得不到提升，以后没有办法抓住更好的机会。

想听听名哥的意见，我是还应该继续坚持吗？坚持的话发展方向在哪里呢？

hi，好久不见。我在看机会，但还没有动，微软只是一个目标，成不成还不清楚，主要是想看外地的机会。
感谢你的信任，写了这么长的信息，我准备一下，晚点再给你答复你提到的问题哈。

感觉，你现在的问题有两个方面，一个是对于现状的不满，另一个是对于未来的迷茫。

对于现状的不满是：
1. 工作环境：加班多，团队实力小，业务规模小（甚至还不成熟？）
2. 工作内容：
    - 与自己期望不同：
         - 目前以前端样式为主，你个人期望是后端开发？
         - 数据及用户规模太小，你个人期望大并发/大数据？
         - 太多杂事。这里要看具体是什么杂事

对未来的迷茫：
1. 不知道自己想要什么
2. 不知道自己的职业方向和目标是什么

打算给你回复一篇长长的内容的,我一点点说，可能回复会有些慢，请见谅。
你先回家做自己的事情，晚点或者明天再看，别耽误作息时间。


### 个人建议

1. 关于对于未来/职业的规划

  先说大的方向吧。

从我个人的过去经历来看，说实话我不是一个正面示例，我觉得自己
是把一副好牌给打烂的人。不过，经历过一些错误，看到过一些正面
的例子，期望我的一些想法能给你一些启示。

  我觉得职业规划是件很难把握的事情，尤其是有两点，一是在于对
于目标的规划，二是对于规划的执行。
  对未来的规划是非常难的事情，基于现在的认知对于未来的环境、
自身（能力、状态）和自身与环境的关系（社交关系和资源等）预测，
并在这个预测上建立计划，实在是太多不确定因素了。
  另外，就在于执行层面的事情，个人因素当然是其中的重要原因，
这个就不细说了，另外，外部因素也很重要。打个比方，某个人愿望
是作为行业 A 的顶级专家，但是在他职业生涯初期，这个行业已经被
另一个行业 B 给终结了，他可能出于生活压力而改换行业。

  我自己和周围的朋友和我所了解的一些人，现有的成就与当初设定
的目标可能完全不同，走着走着，可能就偏离了既定的路线，或者是
发现了另一条岔路，越走越歪。但走歪的不一定是过得不好的，只是
回过头看，现在达到的目标，与当初既定的目标，可能差距万里（这个
差距不一定高矮的差距，更多是指类别、方向的不同）

  说了这些，其实想表明以下几点：
  - 职业规划本身是件很难、很多不确定的事情，因此，感到迷茫，或者
    有时感觉到困惑，是很自然的事情。
  - 虽然很困难，但是我们还是得有个大概的目标，这样至少有个方向。
  - 你的职业目标可能要适当地根据自己和世界的变化，不断调整。
  - 短期的目标最容易把控和执行，因此，重点在于短期目标制订和执行；
    中长期的目标，有个大概方向就好，甚至不用太过严肃，但得有那么一个,
    称为理想也可以。

  那么，我要强调的，其实就是短期目标/规划和执行，个人建议是 1 年以内，
这个是相对比较好把控的。太长久的目标，说实话，谁都无法笃定。
  毕竟，回过头对历史进行分析，大家都能说的头头是道，但历史就是历史，
说得再多它也无法改变；对于未来，谁敢确信地给予准确答复呢?

  至于具体的目标，我无法给予你建议，我个人看好的，不一定是你感
兴趣的，甚至不一定是好的方向。我建议你自己多去尝试和了解，走出自己的
圈子，参加外部的活动、读书、结识朋友等等。

  另外，在关注时事和趋势的时候，警惕各种报刊文章（包括博客、论坛、微博，
朋友圈）还有朋友同事的聊天的影响，要有自己的独立思考的习惯，把当下趋势
和热点当做自己的未来目标、当做自己的“爱好”或“理想” 不一定是合理的事情，
同理，把当前工资高的工作内容当做自己的目标，也不一定是合理的事情。
  换句话来说，纯粹 “面向工资” 或者 “面向热点” 工作，那可能会容易让你
焦虑和失去理智。

  还是那句话，短期目标可能详细和认真一些，中长期的目标需要有一个，
可以根据自己的情况适时变更，但不必给予自己太多的期望与压力。


2. 关于现状的不满及应对方法

2.1 对于工作环境现状的思考与改变：
 - 加班多：这一点，如果是制度上的，那确实短期难办。
           但如果是因为效率问题还是管理问题导致的，是否能有自己
           改善或者推动的余地？比如，提高工具和流程运转的效率？比如，
           与领导沟通甚至提供优化方案？
           如果你愿意的话，甚至可以以管理层的角度去思考和优化，至少，
           可以培养和拓宽自己的思路。
  - 团队实力：这一点样，可能你自己难以改变。但可以想办法主动提升
              自己和团队
  - 业务规模：这一点估计难办，但可以对标业界领头羊来学习现有的最佳实践。

  总之，对于环境的改变，确实相对困难些。尽可能自己提升自己，想办法
  去大公司，尤其是大公司，行业、业务相近的可能更容易跳槽，
  你的经验价值更有持续性。

2.2 对于工作内容的思考与改变：
  - 前端工作： 
        - 其实前端工作也是有深度的，比如面向图形、编译、图像、声音、网络
          处理的。而且，许多知识和应用其实跟后端是有关的。
           - 前端涉及的通信协议(HTTP/1.1/2/3, WebRTC, JSON) 等；
           - 前端涉及的存储：LevelDB, LocalStorage
           - 前端涉及的语言及编译工具：Javascript, WASM, Babel, Typescript 及其他
           - 前端涉及的并发控制：Javascript 的异步处理/并发处理
           - 前端的安全控制
        所有这些概念、知识和技术的理解，对于后端的理解是大有益处的。
        如果你后续想做架构师、技术管理，那么这些也是挺重要的知识的。

  - 数据规模：
        - 规模上不去，确实在工作上你是改变不了的，但是自己要学习还是
          有足够的方向和资料的：
            - 对于工作中用到的 MySQL/Redis 的方式，是否足够深入？对于
              业界标杆的最佳实践的文章是否有通度过？
            - 


        - 如果自己目标是大数据/大并发，那么，你现在私下里，是否有去
          准备和了解相关的知识（比如，通读各大厂商的相关文章/论文，
          通读其中的概念知识，搭建学习的平台？）

   - 太多杂事：
        - 这个要结合自己的具体事务来分析了：
            - 是否能自己单独提升效率的，比如改进工具或者改进使用工具的方法
              和流程
            - 是否能和同事一起提升和改进效率的
            - 是否能和领导一起想办法提升和改进效率的


总之，对于现状：
1. 建议与优秀的团队一起工作，个人想改变环境是想对困难的, 在优秀的公司和
   团队里工作，它能提高你的视野和职业素养的上限。(但注意，与优秀的人共事，
   有时候你的职业上限不一定会更容易)
2. 在能达到 1 之前，请安心提升自己，和尽力改变现状环境, 努力提升自己和
   环境的人，总会提现出他的价值的。而且，说不定现有公司会给你更大的回报。
